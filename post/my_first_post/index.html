用 Hugo 搭建自己的 blog —— My first post

11 May 2020, 19:26
踩坑记录
hugo

这是第一篇文章。算是记录一下自己用hugo搭建博客所遇到的问题吧，我是根据🐑哥的教程来搭建的。 https://www.bilibili.com/video/BV1q4411i7gL?from=search&seid=3132796518941291439

我搭建博客所使用的环境是 macOS（ Catalina 10.15.4 ），并提前安装了 brew工具 和 git ，且配置好了 git 的 username 和 email ；

首先， 我们来梳理一下用hugo搭建博客的流程：

1.安装 Hugo（我使用的是brew工具进行的安装）；

brew install hugo

2.用 Hugo 创建新的（博客）站点页面；

hugo new site myblog                                     //my blog 为自定义名称

3.下载并设置主题；

进入myblog根目录下，输入

git clone https://github.com/yoshiharuyamashita/blackburn themes/blackburn          // 此处的themes/blackburn的功能是建立文件夹

4.在本地启动服务器测试博客的效果；

hugo server -t blackburn --buildDrafts                  // -t 代表指定主题 blackburn是我下的主题的名字，可替换成你下载的主题名称

5.创建一篇新的文章，并使用 Visual Studio Code进行编辑；

hugo new post/my first post.md                          // post/my first post.md 为自定义的新文件夹与其根目录下的文件；另，我本想用"posts"而非"post"，但该主题似乎制定路径为"content/post"。顺带一提，创建的文件都可以去finder中确认；

6.在github 创建一个新的 repository,并将本地博客部署到 github 的 repository 里;

hugo --theme=blackburn --baseUrl="https://muggle110.github.io/" --buildDrafts             // 双引号内为你的 repository地址。如果成功了可以在 myblog 根目录下看到生成了一个 public 文件夹；

7.为本地的 public 文件夹创建一个 .git 仓库，并将 public 目录下所有文件添加到git仓库内；再提交至 github 服务器

git init                                                // 初始化一个空的 git仓库
git add .                                               // 全选并添加当前目录下所有文件
git commit -m "my first post"                           // 将添加的文件提交至仓库并添加说明"my first post"

8.将本地的 public 文件夹与 github 的 repository 进行关联，并将本地的.git部署到互联网上；

git remote add origin https://github.com/muggle110/muggle110.github.io.git           //注意最后有一个 .git
git push -u origin master                               //当出现 * [new branch]      master -> master ，说明推送成功了；

CAUTION ———— 一些踩坑的经验

  1. 当使用 hugo server -D 或 hugo server -t blackburn –buildDrafts 来在本地检查页面时， 必须得是在 /myblog 的根目录下输入命令，否则会出错；

  2. 当使用 git add . 或 git add -A 或 git commit -m “my first post” 来添加至git时， 必须在 /myblog/public 的根目录下,否则会出错；
