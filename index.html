hugo new post/x x x.md 无法正确生成.md文件

120 May 30300, 00:00
踩坑记录
hugo

问题描述： 使用 hugo new post/my post 05 03 2020.md 新建 markdown 文件， 结果在 content/post/ 生成了一个文件名为 “new” 的 非.md 的空白文件； 解决方法： 新建.md文档时文件名不要用空格，可使用下划线代替，即可正确生成。正确的格式应该是： hugo new post/my_post_05_30_2020.md

hugo

这是第一篇文章。算是记录一下自己用hugo搭建博客所遇到的问题吧，我是根据🐑哥的教程来搭建的。 https://www.bilibili.com/video/BV1q4411i7gL?from=search&seid=3132796518941291439 我搭建博客所使用的环境是 macOS（ Catalina 10.15.4 ），并提前安装了 brew工具 和 git ，且配置好了 git 的 username 和 email ； 首先， 我们来梳理一下用hugo搭建博客的流程： 1.安装 Hugo（我使用的是brew工具进行的安装）； brew install hugo 2.用 Hugo 创建新的（博客）站点页面； hugo new site myblog //my blog 为自定义名称 3.下载并设置主题； 进入myblog根目录下，输入 git clone https://github.com/yoshiharuyamashita/blackburn themes/blackburn // 此处的themes/blackburn的功能是建立文件夹 4.在本地启动服务器测试博客的效果； hugo server -t blackburn --buildDrafts // -t 代表指定主题 blackburn是我下的主题的名字，可替换成你下载的主题名称 5.创建一篇新的文章，并使用 Visual Studio Code进行编辑； hugo new post/my first post.md // post/my first post.md 为自定义的新文件夹与其根目录下的文件；另，我本想用"posts"而非"post"，但该主题似乎制定路径为"content/post"。顺带一提，创建的文件都可以去finder中确认； 6.在github 创建一个新的 repository,并将本地博客部署到 github 的 repository 里;
